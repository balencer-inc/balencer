# brand-lint（対外文書の自動検品）

CLAUDE.md 1章「やらないこと・使わない言葉」と修正学習の蓄積を機械チェックに落とした検品工程。
**対外に出る成果物を作ったら、報告前に必ずこれを通す。** 各担当（dev/marketing/creative/consulting/finance）共通の最終工程。

## 実行方法

```bash
# 通常文書（LP・提案書・note・スライド原稿）
python3 scripts/brand_lint.py <ファイル...>

# SNS下書き（ハッシュタグ・問いかけ連続の追加チェック）
python3 scripts/brand_lint.py --sns <ファイル...>
```

終了コード: 0=問題なし / 1=ERRORあり / 2=使い方エラー

## チェック内容

| レベル | 内容 |
|---|---|
| ERROR | 旧表現（デザインコンサルティングカンパニー/経営者の右腕を外注/サービス3カテゴリ/Entryプラン/AI顧問/1000社/オフショア300名） |
| ERROR | 全角ダブルクォート“”（「」に置換するか外す） |
| ERROR | 「対話もできて、コードも書ける」型・「〜力×〜力」掛け算コピー（社内説明用に限定） |
| ERROR | (SNSモード) ハッシュタグ |
| WARN | 「〜に、〜を。」型のAI読点コピーの疑い |
| WARN | タイトル/キャッチコピー（HTML h1〜h3・Markdown見出し）内の読点「、」（改行・句点・スペースで区切る） |
| WARN | 見出しの一部だけ黄色ハイライト/下線を当てる部分強調（`class="hl"`・黄色linear-gradient。外すか全体で統一） |
| WARN | 対外文書の敬称「さん」（クライアント名は「様」。たくさん/みなさん等は除外） |
| WARN | 64社以外の「N社」実績数字（正は64社・生涯4.03億） |
| WARN | (SNSモード) 問いかけの連続 |

## 運用ルール

1. **対象は対外成果物のみ。** `docs/_knowledge/` や CLAUDE.md などの社内文書はNGワード自体を列挙しているため検品対象外。
2. ERRORは修正必須。修正して再実行し、OKになってから阿部さんに報告する。
3. WARNは機械では判断できない項目。該当行を阿部さんに見せて判断を仰ぐ（勝手に握りつぶさない）。
4. 誤検知が続く・新しいNG表現が決まった場合は `scripts/brand_lint.py` のルール定義を更新し、`docs/_knowledge/creative.md` に記録する（2回ミスでルール化）。
5. HTMLもそのまま渡してよい（表示テキスト中の違反を検出する）。

## 新ルールの追加方法

`scripts/brand_lint.py` の先頭にあるルール定義（`FORBIDDEN_PHRASES` / `INTERNAL_ONLY_PATTERNS` 等）に1行追加するだけ。追加したらこのSKILL.mdの表も更新する。
