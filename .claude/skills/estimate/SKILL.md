# estimate（見積書の半自動作成）

見積作成のたびに思い出していた型を固定化したスキル。成果物は **HTML＋CSV＋PDF の3点セット**。

## 0. 最初に確認すること（聞かずに進めない）

金額・商品名・値引きは阿部さんの確定が必須（automode中でも止まる）。以下をヒアリング:

1. **宛先**: 顧客名・担当者名（提案資料に載せる場合は匿名化の要否も）
2. **案件**: 何を提供するか・背景（議事録や提案書があれば場所を聞いて読む）
3. **金額感**: 阿部さんの頭にある水準（税抜/税込も）
4. **相手の言葉**: 相手のHP・資料から引用できるコピーがあるか（信頼感の核）
5. **納期・支払条件**: 特記があるか

## 1. スタイル原則（feedback_estimate_format 準拠）

- **金額は後出し** — 冒頭は課題整理と提供価値。金額表は後半に置く
- **本命は1本** — 松竹梅で散らさない。本命プラン1本＋必要ならオプション別建て
- **値引きは理由付き名称** — 単なる「お値引き」ではなく「ショーケース協力割」など理由が伝わる名前にする
- **相手のビジネスを否定しない** — 現状課題は謙虚に、診断型で書く
- **相手のコピーを引用する** — 相手のHPの言葉を見出しや前文に織り込む
- **提案資料では社名匿名化** — 実績紹介に他社を出す時は業種表記にする

## 2. 作成手順

1. **雛形**: `docs/estimates/nailit-2026/nailit-estimate.html` を最新の見本としてコピーし、`docs/estimates/<顧客ローマ字>-<年>/` を新設して配置する
2. **デザイン**: バレンサーDS v2.0（ローズゴールド）を適用。ロゴ・タグライン "for loved ones." を維持
3. **CSV**: 明細（項目/数量/単価/金額/備考）を同フォルダに `<顧客>-estimate.csv` で出力（BOMなしUTF-8）
4. **検算**: 明細合計・税額・値引き後金額をPythonで検算し、結果を報告に含める（桁ミス防止。feedback_finance_sheet_verification 準拠）
5. **PDF化**:
   ```bash
   "/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
     --headless --print-to-pdf="<出力パス>.pdf" --no-pdf-header-footer "<HTMLパス>"
   ```
6. **検品**: `python3 scripts/brand_lint.py <HTMLパス>` を通す（ERROR 0件まで）
7. **受け渡し**: 阿部さんがすぐ使えるようデスクトップにPDFをコピーする（Finder探索が苦手なため）

## 3. 完了時

- 金額・項目の最終確認を阿部さんに取る（送付は阿部さんが行う）
- 案件の背景・決定・却下案を `docs/_knowledge/finance.md` に追記
- コミット＆プッシュ（「コミット＆プッシュします」と一言告げてから）
