# フルテロップツール作成（伴走スキル）

**喋っている動画にAIが全字幕（フルテロップ）を付けてくれるツール**を、コードが書けないユーザーのPCに作り上げるための伴走スキル。
元ネタは「フルテロップツール作成キット」（フリーランスの学校・しゅうへい氏／2026-06-11）。**正本は [resources/開発手順書.md](resources/開発手順書.md)** で、ツールの全仕様・手順・つまずき対処が入っている。

> 迷ったら会話で終わらせず、開発手順書に沿って手を動かす。ユーザーはコードが書けない前提で、専門用語を避け、日本語でやさしく、コピペで実行できるコマンドと一緒に進める。

---

## 最初にやること

1. **[resources/開発手順書.md](resources/開発手順書.md) を必ず最初に通読**する（これが正本。機能要件表・段階的な作り方・つまずき対処がすべてここにある）。
2. ユーザーに「これからフルテロップツールを一緒に作りますね。まず5つだけ質問させてください」と伝える。
3. 下の**オンボーディング5問を、1つずつ**聞く（まとめて聞かない。1問ずつ回答を待つ）。
4. 回答に合わせて構成を決め、**Day1のMVP（動画→文字起こし）から段階的に**作る。
5. 各ステップで、必要なインストールやコマンドを**コピペできる形**で提示し、**エラーが出たら「何が起きたか／なぜか／どう直すか」をセットで**やさしく説明する。

---

## オンボーディング5問（1つずつ聞く）

- **Q1. 使うパソコンは？** Apple Silicon Mac（M1〜M4）/ Intel Mac / Windows
  - → Apple Siliconなら `mlx-whisper`、Intel Mac・Windowsなら `faster-whisper`。判定は `uname -m`（arm64=Apple Silicon）。
- **Q2. 仕上げに使う編集ソフトは？** Final Cut Pro / Premiere / DaVinci / CapCut
  - → Final Cut Proなら **FCPXML** 出力。それ以外（Mac以外含む）は **SRT** 出力を主にする。
- **Q3. 主に作る動画の向きは？** 縦ショート / 横YouTube / 両方
  - → 文字数・字幕位置のデフォルトを変える（縦15文字前後/中央やや下、横20文字前後）。
- **Q4. テロップの整え具合は？** しっかり整える / ほどほど / ほぼそのまま
  - → フィラー（口グセ）削除の強さを決める。残したい口グセ（なんか／もう 等）は除外設定にできる。
- **Q5. よく誤変換される固有名詞は？** 自分の名前・サービス名・英単語など
  - → 語彙辞書（F8）の初期登録に使う。

## 作る順番（1日1ステップでOK・初日は15分）

開発手順書の「機能要件表」「段階的な作り方」に沿って進める。

1. **MVP（F1〜F5）**：動画を渡す→ローカルWhisperで文字起こし（**単語ごとの時刻=word timestampも取得**）→字幕単位に自動分割→ブラウザ校閲（`http://localhost:5050`）→ **SRT書き出し**。
   - Day1ゴールは「動画パスを渡したら日本語の文字起こしが出る」だけ。完璧にしない。
2. **F6 🎯 音に合わせる（本丸）**：字幕テキストとWhisperの単語時刻を**1文字ずつ系列アライメント（Needleman-Wunsch方式）**で突き合わせ、各行の頭を実際の発話開始時刻にスナップ。ボタン1つで全行再スナップ。
3. **F7〜F10**：賢い分割（語頭/無音に吸着）/ 語彙辞書 / フィラー削除 / 位置調整。欲しいものから足す。

> 作業フォルダはユーザーに断ってから決める。手順書の例は `~/telop-tool/`（scripts/ editor/ resources/）。一時作業フォルダ（Mac/Linux=`/tmp/telop/<stem>/`、Windows=`%TEMP%\telop\<stem>\` か永続フォルダ）も最初に決める。

---

## 実装の必須ポイント（手を抜かない）

- **F2**：Whisperは必ず **word timestamp付き**（`--word_timestamps True` 相当）。これが無いと🎯音合わせの精度が大幅に落ちる。
- **F6**：**Needleman-Wunsch系列アライメント・語頭スナップ**で実装。**文字数比で時間を按分しない。無音検出（VAD）方式も使わない**（一気喋りで誤動作するため）。必ず単語の実時刻基準。
- **縦動画(rotation付き)**：iPhone縦動画は `rotation`/`rotate` タグで縦横が別管理。ffprobeで確認し、自動判定を入れる。
- **書き出し位置**：Mac=`~/Desktop/`、Windows=`%USERPROFILE%\Desktop\`。FCPXMLは `fcp_height`（FCPで編集する実解像度の高さ）に注意（手順書のパラメータ早見表参照）。
- **校閲サーバー起動**：Claude Codeのサンドボックス（preview_start）は使わず、ふつうに `python3 .../server.py --port 5050` で起動し、ブラウザで `http://localhost:5050` を開いてもらう。
- 環境（容量・PATH・Python複数版・Whisper種別）でつまずいたら、開発手順書の「環境構築：躓きポイント注釈」「つまずきポイントと対処」を見て対応する。

## 進め方のトーン

- 1ステップごとに「動いたか」を確認してから次へ。完璧主義で止めない。
- 専門用語が必要なときは一言で補足する。
- エラーは「何が起きたか／なぜか／どう直すか」をセットでやさしく。
- 動くものが1つできたら一緒に喜ぶ。楽しく続けてもらうのが最優先。
