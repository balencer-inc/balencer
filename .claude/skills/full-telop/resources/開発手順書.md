## はじめに

こんにちは、しゅうへいです。

先に言っておきます。ぼくはコードが書けません。プログラミング経験もゼロです。
それでもこのツールは動いていて、毎日ぼくのリール制作を回してくれています。

なぜ作れたか。**「自分で書く」んじゃなくて「AIに要件を渡す」から**です。

Claude Codeに日本語で「こんなものを作ってほしい」と伝えるだけ。
このガイドは、その"渡す中身"＝要件定義をまるごと言語化して、さらに**あなたのPCで動かせるように汎用化**したものです。
ぼくの環境（Mac・特定のフォルダ構成）に縛られた記述を全部取り除いて、Mac/Windows両方で最初から使えるように書き直しています。

## 完成図（処理フロー）

```
[1] 動画ファイルを渡す（.mov / .mp4 / .m4a）
        ↓
[2] ローカルWhisperで音声を文字起こし（単語ごとの時刻も取得）
        ↓
[3] テキストを「読みやすい単位」に自動分割（縦15文字・横20文字前後）
        ↓
[4] ブラウザ校閲画面で確認・修正（分割 / 統合 / 誤字直し）
        ↓
[5] 🎯「音に合わせる」ボタンで全字幕のタイミングを一括スナップ
        ↓
[6] 編集ソフト用ファイルを書き出し（FCPXML / SRT）
        ↓
[7] Final Cut Pro / Premiere / CapCut 等にインポートして完成
```

[5]の「音に合わせる」が既存ツールと最も差がつく心臓部です。
詳しくは後の「機能要件」で説明します。

## 必要なもの・前提環境

### ソフトウェア一覧

| 役割 | Mac（Apple Silicon M1以降） | Mac（Intel）/ Windows |
|---|---|---|
| 文字起こしエンジン | `mlx-whisper`（Apple Siliconに最適化・高速） | `faster-whisper`（CPU/GPU汎用） |
| 音声解析 | `ffmpeg` / `ffprobe` | 同左 |
| 校閲サーバー | Python（Flask） | 同左 |
| プログラミング言語 | Python 3.9以上（3.10+推奨） | 同左 |
| フォント | Apple SD Gothic Neo（OS付属）または NotoSansJP-ExtraBold | NotoSansJP-ExtraBold.ttfを別途ダウンロード |
| 仕上げ編集ソフト | Final Cut Pro（FCPXML対応） | Premiere / DaVinci / CapCut（SRTで読み込み） |
| 画像ライブラリ | Pillow（焼き込みプレビュー時のみ・任意） | 同左 |

> **Apple Siliconとは？** M1、M2、M3、M4チップを搭載したMacのことです。2020年末以降に発売された多くのMacが該当します。「このMacについて」→「チップ」の項目で確認できます。Intel Macには `faster-whisper` を使います。

### 用意する作業フォルダ（例）

このガイドでは **`~/telop-tool/`** を作業場所の例として使います。
好きな場所（デスクトップ・ドキュメント等）に作っても構いません。その場合はコマンド内のパスを読み替えてください。

```
~/telop-tool/
  scripts/            ← 文字起こしスクリプト等
  editor/             ← ブラウザ校閲エディタ
  resources/          ← 語彙辞書
```

## 環境構築：躓きポイント注釈

### PATHが通らない（コマンドが見つからない）

```
zsh: command not found: ffmpeg
```
このエラーが出たら、PATHが通っていません。

**Mac の対処:**
```bash
# .zshrc に追記（Homebrewのパスを通す）
# Apple Silicon Mac の場合
echo 'export PATH="$(brew --prefix)/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

**Windows の対処:**
- 「システムのプロパティ」→「環境変数」→「Path」に ffmpeg の bin フォルダを追加
- PowerShell を再起動して再確認

### Apple Silicon 判定の確認

```bash
# Mac でチップを確認
uname -m
# arm64 と出れば Apple Silicon → mlx-whisper を使う
# x86_64 と出れば Intel Mac → faster-whisper を使う
```

### mlx-whisper が Intel Mac で入らない

mlx-whisper は Apple Silicon 専用です。Intel Mac では `faster-whisper` を使ってください。Claude Code に「ぼくの環境は Intel Mac なので faster-whisper で作ってほしい」と伝えるだけで対応してくれます。

### ffmpeg が見つからない（Windows）

winget でインストール後もパスが通らない場合は、[ffmpeg公式サイト](https://ffmpeg.org/download.html) からzipをDLして、`C:\ffmpeg\bin` に解凍後、システムのPATH環境変数に `C:\ffmpeg\bin` を追加してください。

### Python が複数バージョン入っている

Macで `python3` と `python3.11` が混在している場合、インストールしたパッケージが正しいバージョンに入っているか確認が必要です。Claude Code に「python3.11 を使って仮想環境（venv）を作って、その中で全部入れてほしい」とお願いするのがスッキリします。

## 機能要件表

### 4-1. MVP（まずここまで作る）

| # | 機能 | 要件 |
|---|---|---|
| F1 | 動画の受け取り | 動画ファイルパスを渡すと処理が始まる（.mov / .mp4 / .m4a 対応） |
| F2 | 文字起こし | **ローカルのWhisper**で音声→テキスト。日本語対応。単語ごとの時刻（word timestamp）も取得する |
| F3 | 自動分割 | テキストを字幕単位に分割。**縦15文字・横20文字前後**、読みやすさ（1秒あたり文字数）で調整 |
| F4 | 校閲UI | ブラウザ（`http://localhost:5050`）で字幕一覧を表示。各行を**分割・統合・テキスト修正**できる |
| F5 | 書き出し | まずは**SRT**（全ソフト共通）で書き出し。慣れたら**FCPXML**（Final Cut Pro専用） |

### 4-2. 差がつく機能（MVPの次に足す）

| # | 機能 | 要件 |
|---|---|---|
| F6 | 🎯 音に合わせる | ボタン1つで**全字幕のタイミングを音声に再スナップ**。**文字数比で割らず**、Whisperの単語時刻を使い、字幕テキストと音声を1文字ずつ突き合わせる（系列アライメント/Needleman-Wunsch方式）。各行の頭を**実際の発話開始時刻**に合わせる。文言を書き換えても、語を足しても落としてもズレない |
| F7 | 賢い分割 | ✂︎分割したとき、文字数で割らず**いちばん近い「語の頭」や「間（無音）」に吸着**。句の途中で切れない |
| F8 | 語彙辞書 | 固有名詞・自分の言い回しの**変換ミスを自動補正**（辞書ファイルに登録）。自分の名前・サービス名・英単語のスペース有無等 |
| F9 | フィラー削除 | Q4で選んだ「整え具合」に従って口グセを落とす。残したい口グセ（なんか／もう 等）は残す（除外設定） |
| F10 | 位置調整 | 字幕の縦位置（Y）を調整。YouTube向けは下げる、ショート向けは中央寄せ等 |

> **F6が本丸です。** ぼくも最初は「出だしは合うけど後ろがズレる」で詰まりました。
> 「文字数で時間を按分」をやめて「**単語の実時刻に合わせる**」に変えた瞬間、バチバチに合うようになりました。
> Claude Codeに「文字数比ではなく単語のタイムスタンプ基準で、Needleman-Wunsch系列アライメントで実装して」と強めに指定してください。

## 詳細パイプライン（ツールの内部仕様）

Claude Codeにより詳しく実装させたい場合の参考資料です。各処理の役割と入出力を説明します。

### 処理の流れ（各モジュールの役割）

```
[prep]
動画ファイルを受け取り → Whisperで文字起こし → 生のFCPXML/SRT生成
→ 校閲用JSONファイル(_校閲済み.json)と生テキスト(_raw_text.tsv)を生成

[server]
Flask（port 5050）でブラウザエディタを提供
→ 動画の読み込み・校閲状態の管理・分割/統合の受付・保存のトリガー

[校閲フェーズ（ブラウザ操作）]
ユーザーがブラウザで字幕を確認・修正

[retime / 🎯 音に合わせる]
校閲後の字幕テキストとWhisperの単語時刻データを突き合わせ
→ 各字幕行の開始時刻を実際の発話頭に再スナップ（Needleman-Wunsch系列アライメント）

[reapply]
修正済み字幕JSONをFCPXMLテンプレートに適用

[embed]
フォント・サイズ・位置情報をFCPXMLに埋め込み
→ デスクトップに最終FCPXMLを出力（Windows/Mac共通: ~/Desktop/ または %USERPROFILE%\Desktop\）
```

### 中間ファイルの構成（作業フォルダの例）

```
~/telop-tool/work/<stem>/
  <stem>.mov（または.mp4）    ← 元動画の作業コピー
  <stem>.fcpxml               ← Whisper出力（生）
  <stem>_pipeline.fcpxml      ← 編集元テンプレート（不変・正本）
  <stem>_raw_text.tsv         ← 生の文字起こし（学習比較用）
  <stem>_校閲済み.json        ← 編集対象（ブラウザで修正した内容が入る）
  .telop_post_hook.json       ← フォント・位置・出力先の設定

# Mac/Linux の一時フォルダ案: /tmp/telop/<stem>/
# Windows の一時フォルダ案: %TEMP%\telop\<stem>\
```

> **一時フォルダについて**：Mac/Linuxでは `/tmp/` が一般的です（再起動で消えます）。Windowsでは `%TEMP%\telop\` を使うか、`~/telop-tool/work/` のような永続フォルダにまとめるとわかりやすいです。Claude Codeに「一時作業フォルダはどこにするか」を最初に決めてもらうのが確実です。

### 校閲済みJSONのスキーマ（Claude Codeに共有する場合）

```json
{
  "version": 1,
  "stem": "<動画ファイル名（拡張子なし）>",
  "items": [
    {"start": 0.0, "end": 2.0, "text": "字幕テキスト", "src": 0}
  ]
}
```

- `start` / `end`：秒単位の表示時刻
- `src`：元の文字起こし（Whisper出力）のインデックス。分割しても同じ番号を共有することで、学習・ロールバックに使える

## パラメータ早見表（フォント・位置調整）

字幕のサイズや位置を調整するときの参考値です。
Claude Codeに「縦ショート用に設定して」と伝えるだけでも対応してくれます。

| 用途 | フォント | サイズ(fontSize) | baseline | 位置Y指定 | fcp_height |
|---|---|---:|---:|---|---:|
| 縦ショート 2160×3840 | NotoSansJP ExtraBold | 31 | -54相当 | 0 3.75（中央やや下） | 3840 |
| 横YouTube（高さ540基準） | 好みのゴシック体 | 35 | -54 | UI位置Y | 省略可 |
| 横YouTube 4K（高さ2160） | 好みのゴシック体 | 140 | -216 | 用途次第 | 2160 必須 |
| 低解像プロキシ | 用途準拠 | 用途準拠 | 用途準拠 | 用途準拠 | FCPで編集する実際の高さ |

**原則式:**
- `fontSize = 動画の高さ × 0.065`
- `baseline = 動画の高さ × 0.10`（マイナス値）
- `fcp_height = Final Cut Proで編集するときの解像度の高さ`

> **Windowsユーザーへ：** Final Cut ProはMac専用です。Premiereの場合は FCPXML ではなく **SRT** で書き出して読み込んでください。DaVinci Resolve・CapCut も SRT に対応しています。

## つまずきポイントと対処

### 音ズレ・タイミング関連

| 症状 | 原因 | 対処 |
|---|---|---|
| 字幕が音より**早く出てズレる**（全体的に） | 文字数比での按分になっている | F6実装時に「単語の実時刻基準・系列アライメント」と強めに指定。Claude Codeに「後ろがズレてる」と繰り返し伝える |
| 🎯 を押すと**全体が大きくズレた** | 無音検出（VAD）方式を使っている場合、一気喋り動画で誤動作する | F6は「Needleman-Wunsch系列アライメント・語頭スナップ」で実装するよう指定。VAD方式（無音検出）は使わない |
| ✂︎ **分割で句の途中で切れる** | 文字数で時刻を按分している | F7実装：分割点を「最も近い語の頭・無音箇所」に吸着させる |

### 文字起こし・変換関連

| 症状 | 対処 |
|---|---|
| 固有名詞・名前が**毎回誤変換**される | 語彙辞書（F8）に登録して自動補正。辞書ファイルは `~/telop-tool/resources/vocabulary.json` に |
| 文字起こしが**遅い・止まる** | Apple SiliconならMLXの設定確認。IntelやWindowsなら faster-whisper で動画の1/3〜1/5の時間が目安 |
| word JSON（単語時刻データ）が**生成されていない** | `--word_timestamps True` オプションが付いているか確認。このデータがないと 🎯 音合わせの精度が大幅に落ちる |

### 設定・環境関連

| 症状 | 対処 |
|---|---|
| 容量が足りなくなる | CoworkではなくClaude Code（ローカル環境）で作る。モデルは約1.5GB。空き容量5GB以上を確保してから始める |
| `flask` が見つからないエラー | `pip3 install "flask>=2.3"` を再実行。`pip` と `pip3` を間違えていないか確認 |
| FCPXMLをFCPに読み込んだら**テロップ位置がズレる** | `fcp_height` の指定が実際の編集解像度と合っていない可能性。4K素材でも低解像プロキシで編集している場合はプロキシの高さを使う |
| FCPXMLのインポートで**DTDエラー** | `<adjust-transform>` の挿入位置がズレている。Claude Codeに「embed_te.pyのadjust-transformの挿入位置を確認して」と依頼 |

### 縦動画（rotation付き）特有の問題

| 症状 | 対処 |
|---|---|
| 動画の縦横が**逆に判定される** | iPhoneで撮った縦動画はrotation情報が別に入っている。ffprobeの `rotation` か `rotate` タグを確認。transcribe_telopスクリプトに自動判定を入れるよう依頼 |

## ここから先：ツールを仕事にする

このツール、自分用に終わらせるのはもったいないです。

世の中の中小企業や個人事業主は、ニッチな自動化ツールを買えません。
汎用ソフトは売ってても、「うちの業務専用」はエンジニアに個別発注＝数百万かかるから。
だから非エンジニアでAIを使える人にチャンスが回ってきます。

- 動画発信している人・企業の**フルテロップ代行**（1本いくら、で受注）
