# ポジショニングマップ / 戦略ビジュアル

コンサル・提案・壁打ちで使う「2軸の戦略マップ」を、**目で見て一瞬で理解できる綺麗さ**で作るスキル。
初出＝Clavis Partners案件のビフォー/アフター ポジショニングマップ（阿部さん評「非常に綺麗」）。この品質を型にして、バレンサーでもTSUGIでも再現する。

正典テンプレ＝`docs/clients/clavis-partners/01_提案/ポジショニングマップ_before-after.html`（実例）／このフォルダの `assets/template.html`（汎用の骨格）。

## いつ使うか
- ポジショニングマップ（単価×関与、価格×品質、など任意2軸）
- ビフォー/アフター比較（同一軸で過去と現在を並べる。10年前の事業計画 vs 現在、など）
- 競合マップ／2x2マトリクス／狙い目ゾーンの可視化
- 提案・議事録・壁打ち補助資料で「文章より図で見せたい」時

## 成果物の原則（崩さない）
1. **自己完結HTML**：外部CDN・Webフォント・画像URLを使わない。フォントはシステムスタック（明朝×ゴシック）。SVGはインラインで手書き。→ Artifactやメール添付でそのまま開ける。
2. **テーマ対応**：`prefers-color-scheme`＋`data-theme`の両対応。トークン（CSS変数）で色を定義し、ライト/ダーク両方をちゃんと設計する（単純反転しない）。
3. **同一軸で比較**：ビフォー/アフターは必ず縦横の軸を揃える。軸を変えると比較にならない（＝この型の価値が消える）。
4. **保存先**：顧客案件なら `docs/clients/<顧客>/01_提案/` 等の正しい場所へ。作ったら顧客READMEにリンクを1行足す。
5. **見せ方**：クリックで見たい要望には Artifact 公開（既定プライベート＝本人しか見られない。共有はページの共有メニューから）。

## ブランド規律（バレンサー）
- 白ベース × 墨（ink）× くすんだブルー。地の色は無彩色でなくアクセント寄りに僅かに寄せる（`#F6F8FA`等）。
- **蛍光イエローのマーカー風下線は使わない**（AIっぽい）。強調は太字・余白・色階層で。
- 見出し＝明朝（編集的・落ち着き）、本文＝ゴシック。CFO同席など堅い場では墨×くすんだブルーで一層抑える。
- ビフォー＝退色グレー（過去）、アフター＝ブルー（現在/live）、狙い目＝抑えた金（`#B8863B`／蛍光でない）。この3色分担が「時間の流れ」を伝える。
- **TSUGIで使う時**：アクセント（`--blue`系）をTSUGIのブランド色に差し替える。構造・トークン設計はそのまま流用。

## 作り方（手順）
1. **軸を決める**：縦軸・横軸に何を置くか（例：縦=単価/Fee、横=関与の深さ）。両端のラベルを言葉で（「助言のみ〜実務まるごと」）。
2. **点を置く**：各プレイヤー/選択肢を (x,y) で配置。座標系は viewBox `0 0 400 380`、プロット域 x=56〜384 / y=30〜336（yは上が高）。
3. **ゾーンを描く**：空白地帯・狙い目・「散らばり」は破線の楕円＋ラベルで。星（★）で狙い目を示す。
4. **読み方を添える**：各図の下に「この図の読み方」を2〜3行の箇条書きで。図だけで終わらせない。
5. **示唆（Takeaway）**：色付きの引用ブロックで「だから何か」を1文。
6. **検品**：`assets/template.html` の骨格を流用し、ダーク/ライト両方で崩れないか・横スクロールが出ないか・全角ダブルクォート混入がないかを確認。対外物なら brand-lint 相当のチェックを最終工程に。

## SVG散布図の座標のコツ
- 高単価＝y小さい（上）、低単価＝y大きい（下）。関与浅い＝x小さい（左）、深い＝x大きい（右）。
- ラベルが点と重なる時は `text-anchor` と xオフセットで逃がす。密集部は線を引かず余白で。
- 点は `<circle r="6">`、ツール群など「面で覆う」表現は小さめ `r="5"` を複数。
- ゾーンは `<ellipse ... stroke-dasharray="5 4" fill="var(--…-wash)">`。

## 品質の分かれ目（ここで差が出る）
- 軸ラベル・凡例・読み方が揃って初めて「一瞬で分かる」。図だけだと自己満足になる。
- ビフォー/アフターは「何が空いていたか→何が埋まったか→どこに残余の勝ち筋があるか」の物語で見せる。
- 色は1色だけ主役（アクセント）。残りは静かに。狙い目の金は1箇所だけに効かせる。
