# weekly-review（週次定点観測）

散らばった案件（クライアント・営業・SNS・補助金・社内開発）を週1回、1枚に集約して「来週の最重要3件」まで落とす業務OS。出力は `docs/_knowledge/weekly-review/YYYY-MM-DD.md` に保存し、蓄積そのものを経営ログにする。

## 手順

### 1. 今週の事実を自動収集（推測より先にログを見る）

```bash
# 今週のコミット（全ブランチ）= AI社員たちの実働ログ
git log --all --since="7 days ago" --pretty="- %ad %s" --date=format:"%m/%d"
# 今週動いたファイル領域
git log --all --since="7 days ago" --name-only --pretty=format: | sort | uniq -c | sort -rn | head -15
```

コネクタが接続されているセッションでは追加で:
- Google Calendar: 来週の予定一覧（`Asia/Tokyo`）
- Gmail: 未返信の重要スレッド
- Notion: タスクDBの期限超過

接続がなくても止まらない。**取れた情報だけで進め、取れなかった項目は「未確認」と明記する。**

### 2. 阿部さんに聞くこと（3問だけ・軽く）

1. 今週いちばん進んだこと／モヤっとしたことは？
2. 営業パイプラインで動きがあった案件は？（新規・返信・失注）
3. 来週、絶対に落とせない予定・締切は？

### 3. レビューシートを作成

`docs/_knowledge/weekly-review/YYYY-MM-DD.md` に以下の型で保存:

```markdown
# 週次レビュー YYYY-MM-DD（第N週）

## 今週やったこと（gitログ＋ヒアリングから）
- ...

## 営業パイプライン
| 案件 | 状態 | 次の一手 | 期限 |
|---|---|---|---|

## SNS・発信
- 投稿本数（X/FB/note）・反応が良かったもの・ネタ台帳の残弾

## お金
- 請求出した/未入金/今月着地見込み（分かる範囲で）

## 詰まっていること・リスク
- ...

## 来週の最重要3件
1. ...
2. ...
3. ...

## ナレッジ棚卸し
- 今週 _knowledge に残すべきだったのに残っていない決定: ...
```

### 4. 締め

- 最重要3件のうちAI社員に任せられるものは、その場で担当スキルに振るか次セッションの指示文を用意する
- コミット＆プッシュ（「コミット＆プッシュします」と一言告げてから）

## 運用ルール

- 1回15分以内で終える設計。完璧な集計より継続を優先
- 数字（売上・請求）は推測で埋めない。不明なら「未確認」と書く
- 4週連続で同じ「詰まり」が残っていたら、構造問題としてスキル化/自動化を提案する（feedback_repetition_to_skill_automation 準拠）
