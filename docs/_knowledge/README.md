# 社内ナレッジベース `docs/_knowledge/`

**新しいセッション（CLI・デスクトップ・クラウド/Web）でも回答の質を落とさないための共有メモリ。**
各PCのグローバルmemoryはクラウドに来ないため、引き継ぐべき知見はここに残す。
各担当（秘書・開発・マーケ・経理・クリエイティブ・コンサル）が、作業で得た知見・決定事項・顧客の好み・命名規則・却下案とその理由を蓄積する。

## ファイル構成
- `company.md` — 会社・サービスの背景（共通前提。詳細の正本は `docs/company/`・`docs/handoff/`）
- `secretary.md` — 秘書の知見・定型対応ログ
- `dev.md` — 設計判断・規約・ハマりどころ
- `marketing.md` — 効いた投稿・施策・NG表現
- `finance.md` — 経理・補助金まわりの知見
- `creative.md` — トンマナ・コピー・素材の決定
- `consulting.md` — 提案・戦略・クライアント分析の知見

## 運用ルール
1. **その場で書く** — 重要な決定や好みが出たら、該当ファイルにすぐ追記。
2. **日付＋要点** — 1エントリは `## YYYY-MM-DD 要点` の形で。
3. **消さず更新** — 古い情報は削除せず「更新」として追記し、流れを残す。
4. **報告に含める** — 作業の最後に「今回 _knowledge に何を残したか」を阿部さんへ報告。
5. **コミットする** — 会話で終わらせない。必ずファイルに残してコミット＆プッシュする。

## このリポジトリの memory との関係
ローカルのグローバルmemory（`~/.claude/projects/.../memory/`）は引き続き各PCの素早いメモとして使う。
ただし**クラウド/Webや他PCにも引き継ぎたい事実は、ここ `docs/_knowledge/` にも残す**のが原則。
