# 社内ナレッジベース `docs/_knowledge/`

**新しいセッションでも回答の質を落とさないための共有メモリ。**
各担当（秘書・Webデザイン・開発・SNS・マーケティング）が、作業で得た知見・決定事項・
顧客の好み・命名規則・却下案とその理由をここに蓄積する。

## ファイル構成
- `company.md` — 会社・プロダクトの背景（共通前提）
- `secretary.md` — 秘書の知見・連絡先・定型対応ログ
- `web-designer.md` — デザイン上の決定・トンマナ・素材
- `developer.md` — 設計判断・規約・ハマりどころ
- `sns.md` — 効いた投稿・NG表現・運用ルール
- `marketing.md` — 効いた施策・訴求・価格反応

## 運用ルール
1. **その場で書く** — 重要な決定や好みが出たら、該当ファイルにすぐ追記。
2. **日付＋要点** — 1エントリは `## YYYY-MM-DD 要点` の形で。
3. **消さず更新** — 古い情報は削除せず「更新」として追記し、流れを残す。
4. **報告に含める** — 作業の最後に「今回 _knowledge に何を残したか」を田部さんへ報告。
5. **コミットする** — 会話で終わらせない。必ずファイルに残してコミットする。
