# MVV策定・浸透ワークの型（プレイブック）

看板1枚目「組織開発・MVV」の実運用テンプレ。旧Notionの実案件から型だけを抽出（社名・個人名は伏せる）。
提案・見積・進行の叩き台に使う。実際の日程・人数はクライアントに合わせて調整する。

## 1. MVV策定プロジェクトの標準フロー（6フェーズ）

**体制の目安**：クライアント側＝経営コア＋選定メンバー数名（事業/部門ごとにチーム分け、計6〜7名）／バレンサー側＝ファシリ・ライティングで3名程度。

| 段 | 内容 | 目安時間 |
|---|---|---|
| ① キックオフ | 目的合わせ。**事前に社内告知**してもらう（会社の方向性をみんなで決めること、決定を基に採用サイト/コーポサイトもリニューアルしうること） | 1〜2h |
| ② 経営者インタビュー | 理念・考えの「骨」をヒアリング。ミッション/ビジョンの核を固める。**個人としてのビジョンや過去の原体験まで**ざっくばらんに引き出す | 1.5〜2.5h |
| ③ 選定メンバーIV（チームA） | 「らしさ」インタビュー。理念の肉付け・再構築。個人ビジョンの確認。**行動指針はメンバーの想いを中心に**作る（1名30分〜1h） | 2h |
| ④ 選定メンバーIV（チームB） | ③と同様。終了後、追加の宿題を出す場合あり | 2h |
| ⑤ プロジェクトメンバーMTG | 全体の目線合わせ・方向性の統一（必要に応じて追加1〜2回） | 2h |
| ⑥ MVV完成 | バレンサー側でライティング。**クレドカード制作はオプション**（用紙代別途） | － |

**その後の接続施策（順不同）**：Notion導入／Google Workspace導入／採用サイト制作／採用媒体の選定・制作／コーポレートサイト制作／写真・動画撮影。
→ MVVは「作って終わり」でなく、CI/VI・制度・採用・発信へ接続する。制作は必ず伴走に挟む（単体受注しない）。

## 2. 課題図書ワーク（浸透・組織開発セッション）の設計

MVV浸透や組織の内的動機づけを高めるための、定期セッションの型。阿部さんのコーチング哲学（人は自分が話すことで自分の答えにたどり着く）と地続き。

### 開催意図
- 課題図書でインプットを揃え、各自が感じた/考えたことをアウトプットする場をつくる＝**考えを説明し伝える訓練**
- 周囲と自分の差を認識し、相互に刺激を与える。刺激への自分の反応を見つめ、柔軟に反応できるようになる

### 図書選定の観点
- 初回は話しやすく、イメージしやすい内容
- テーマに沿いつつ、**「危うさ・現実的な限界」も含む図書**を選ぶ（賛否・葛藤を議論できる余地を残す）。例：内発的動機づけ・自己効力感・リーダーシップ変革・「らしさ」偏重のリスク 等

### 進行の姿勢（ファシリの肝）
- **立場を作らせない**：管理者向けに見える内容でも「協調してやり遂げる時に意識すべきことは立場を問わず同じ」。「自分は立場が違う」という前提こそ視野を狭める。高い視座で捉えてもらう
- **正解を探さない**：もっともらしい意見はあっても正解はない。「難しい・理解できない」も一つの反応で、恥ではない
- 一番大切なのは、**正解のない問いに何を感じたかを相手に伝える説明努力**と、**伝わらない限界を感じること**
- 楽しむイベントではなく、**定期的な訓練＝仕事**として積極姿勢で取り組んでもらう

### 標準の流れ
チェックイン → チーム分け（少人数A/B/C）→ 問いに沿った議論 → 発表・共有。読後アンケート・問いだしは事前に用意する。

## 3. リブランディング社内発表スピーチの型
MVV/リブランディングを社員に腹落ちさせる社内発表の「感情の順序」。数字（売上・還元目標等）はクライアント固有なので入れ替える。**順序が型**。

1. **これまでの歴史の肯定**：顧客に誠実に向き合い続けた／当たり前に見えて実は多くの変化を越えてきた → 心からの敬意
2. **時代認識と危機感**：変化のスピードが加速。テンポが速いとペースが崩れ、回復前に次の波。だから今が変化点
3. **経営者の覚悟＝還元の約束**：待遇を上げたい。そのため原資（売上＝顧客の信頼）を増やす必要がある
4. **新たな柱の必要性**：既存の延長だけでは限界。新事業・新設備・新規顧客との出会い
5. **リブランディングの意義**：全てを変えることではない。温故知新＝本当の価値を磨き直すこと。今が適切なタイミング
6. **社内ワークで見えた真の価値**：本当の価値は社員の向き合い方・誠実さ・文化そのもの＝模倣できない
7. **なぜ外部を入れたか**（肝）：社内だけだと「選ばれた案・選ばれない案」が生まれ率直なFBが難しい。だから外部を入れた。必要な痛みだった
8. **対話継続の約束**：意見を伝えやすい体制、その都度チューニングしながら豊かにする
9. **具体**：売上/還元目標・達成シナリオ・組織変更・各部署への期待
10. **締め**：これまでへの敬意＋「未来はここにいるみんなの手で創られる」

## 4. 年間伴走プログラムの型（CORE/PROJECT）
12ヶ月の伴走プログラムの骨格。**旧設計の「年額2,000万円」「DX・採用・AIを全部やる包括」表現と創作事例は使わない**。金額は現行サービス階段（CORE 月30〜100万／PROJECT）に合わせて都度設計。

- **フェーズ構造（12ヶ月4フェーズ）**：P1(1-2M)理念・MVV再定義／本質価値抽出 → P2(3-5M)ブランド戦略・AI/DXプラン策定 → P3(6-10M)実行支援・クリエイティブ制作・施策展開 → P4(11-12M)成果振り返り・定着化・来期戦略。※制作は必ず伴走に挟む（単体受注しない）
- **開始フロー**：初回ヒアリング/相談 → NDA・契約 → キックオフ（2〜3h）→ 初期診断・ベースライン評価
- **進行管理**：月次定例／週次タスク共有／四半期レビュー（KPI達成度と修正計画）／Notion・Drive共有／緊急対応・追加相談のルール
- **納品物の型**：MVVドキュメント・ブランドガイドライン・AI/DXロードマップ・運用ガイド・コンテンツカレンダー・定例レポート/KPIダッシュボード
- **クライアント準備物**：現行事業/顧客データ・既存理念/経営計画・組織図/評価制度・過去の解析データ・各種アクセス権限（CMS/Analytics/SNS）
- **契約Q&A・リスク条項（そのまま流用価値が高い）**：中途解約・契約変更の取り決め／納期遅延・成果未達時の対応ポリシー／知的財産・制作物の権利帰属／コミュニケーション不調・コンフリクト時の対処
- **アップセルの型**：2年目以降の拡張、成果連動報酬の見直し、幹部研修・エグゼクティブコーチングの追加

## 5. 使うときの注意
- クライアント社名・個人名は出さない（業種表記）。金額・日程は現行のサービス階段（ENTRY／CORE／PROJECT。SOERUは当面控えめ）と料金に合わせて都度設計し、勝手に金額を書かない
- 提案は「課題→戦略→施策→効果」で構成。会社定義・実績（64社・生涯4.03億）の正本は `docs/company/`・`docs/handoff/` から引く
