## 2026-06-16 補助金（見立て／ミタテ）の「余白」と納品コンプラの一線

ファーストプレイス案件起点で、見立て（ミタテ）補助金パッケージを営業のどこで・どう使うか、リスキリング助成との重ね方、「工数が余る分をどう価値化するか」を整理。結論を会社の判断として残す。

### 前提（商品構成・2026-06時点）
- 見立て＝**ソフト150万＋役務（導入設定・カテゴリ6）90万＋保守（カテゴリ7）50万＝約290万**のパッケージ。補助1/2 → 顧客実質約145万。
- ソフトは登録済み。役務・保守は審査中（全部通れば「導入〜定着」まで補助の枠内で見せられる）。
- **枠は通常枠のままで正しい**。インボイス/電子取引・セキュリティ等の特別枠は「その機能を持つツール」が要件で、見立て（経営ボード系）は要件不一致。むやみに特別枠に出すと落ちる。通常枠内の「機能数による上限区分」「賃上げ・リスキリング加点」の最適化で攻めるのが現実的（最新公募要領で要確定）。

### 商品の本音と位置づけ
- 見立ては「登録商品を持っている＝検索ヒット・信頼の見せ方＋予算取り」を主目的に組んだ、いわば**入口づくりのための器**。単体で強い pull 需要のある機能商品ではない。
- よって**ミタテ商品＝出会いの入口／補助金（制度）＝クロージングの武器**（正本 sales-strategy-2026 と一致）。商談で「これ違いますね」となるのは失敗ではなく入口が機能した証拠。無理に補助金を当てにいかず、伴走・ヒトツ・MVV へピボットしてよい。

### 「余白」は3か所にある（すべて表でやれる正規ルート）
1. **ソフトの定義の幅**＝器が自由度高いほど、クライアントが本当に欲しいもの（数字可視化・業務棚卸し・仕組み化）を「登録ツールをその会社向けに構成したもの」として正規に納品できる。
2. **役務90万＝そもそもカスタマイズ予算**。踏み込んだ構成・設定はここで正規にやれる。
3. **リスキリング助成**＝「使いこなしまで」を別経費で重ねる（制度が別なら併用可。同一経費の二重取りのみNG）。
→ 余った工数は「裏で別物」ではなく **①器の構成 ②役務 ③研修 ④自費の別建て/次回申請/伴走** に逃がす。

### 一線（NG）と判定基準
- **「表向き見立てで請求し、裏で登録機能と無関係の別物を納品」はアウト**（虚偽申請・目的外使用＝不正受給。返還＋加算金＋公表、最悪IT導入支援事業者の登録取消＝チャネルごと消滅）。正本 7章-5／4-2-3 とも一致。
- ただし**阿部さんの意図は「器を使って“本当にやりたいこと”の一部でもちゃんと納品物として作る」＝正規ルートそのもの**。やりたいこと/補助金目的かではなく、判定は一点だけ：
  - **セルフチェック：「この納品物は登録ツールの“この機能”をこの会社向けに構成・実装したもの」と検収で一文で言い切れるか。** 言い切れる→全部表でやれる／言い切れない→その部分だけ自費・次回申請に切り出す。
- 運用習慣：申請時の機能説明と納品物の語彙を揃え、「申請＝納品」が書面で辿れるようにしておく。
- 既存ルール再確認：ヒトツと補助金ツールを同一見積に混ぜない（実質値引き＝規約違反リスク）。ヒトツは2〜3ヶ月後に別契約・別見積でアドオン。
