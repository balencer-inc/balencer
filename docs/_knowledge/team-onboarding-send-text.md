## ① メンバーへ送る文面（そのままコピペ）

> ※ 先に「②初回セットアップ」を各人のMacで済ませてから送る。実名・担当は必要に応じて足す。

```
Claude、チームで使い始めます。準備しておいたので、下のとおりやってみてください。

【やること】
1. Macの「Claude」デスクトップアプリを開く
2. バレンサーのチームアカウントでログインしていることを確認（会社の仕事はこのアカウントで）
3. 上の「Code」タブを開くと「balencer」というフォルダが開いています
4. チャット欄に、まずこう打ってみてください：

   /secretary おはようございます。今日から使い始めます。何ができるか教えてください。

これで秘書AIが受付して、できること・使い方を案内してくれます。
ターミナル（黒い画面）は触らなくて大丈夫。ふつうに日本語で話しかけるだけです。

【1つだけ覚えてほしいこと】
作ったもの・決めたことは「会話の中だけ」で終わらせず、箱（balencer）に残します。
Claudeが自動で保存してくれますが、迷ったら「これ残しておいて」と一言添えればOK。
これで、あなたの作業が会社の資産として全員に共有されます。

詳しい入門は、Codeタブで「ONBOARDING.md を読んで案内して」と打つと出てきます。
困ったら何でも /secretary に聞いてください。
```

## ② 初回セットアップ手順（各Macで一度だけ・阿部さん or Claudeが実施）

メンバーはターミナルを触らない前提。以下は**設定担当（阿部さん、または各Macで開いたClaude Code）が一度だけ**行う。

1. **Claudeデスクトップアプリを入れる**（未導入なら claude.ai からダウンロード）
2. **バレンサーのチームアカウントでログイン**（会社利用を集約するため。個人Maxは使わない）
3. **git と gh（GitHub CLI）を入れる**（Macに未導入の場合）
   ```
   brew install git gh
   ```
4. **GitHubにログイン**（このメンバーが balencer リポジトリにアクセスできる状態にしておく）
   ```
   gh auth login
   ```
5. **リポジトリを clone**（置き場所は任意。例：~/Projects）
   ```
   git clone <balencerリポジトリのURL> ~/Projects/balencer
   ```
6. **Claudeデスクトップの「Code」タブで ~/Projects/balencer を開く**
7. 動作確認：チャットに `/secretary こんにちは` と打って、受付が返ればOK

> ポイント：3〜5の技術的な準備を設定担当が済ませておけば、メンバーは以降 Codeタブで日本語を打つだけ。gitコマンドはClaudeが代行するので本人は覚えなくてよい。
