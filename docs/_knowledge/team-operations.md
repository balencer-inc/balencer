## 2026-07-10 チーム運用の型を確定

### 前提の正しい理解（ここを外すと崩れる）
- **チームプランが共有するのは請求と席（アカウント管理）だけ。** スキル・CLAUDE.md・ナレッジは自動共有されない（公式ドキュメントで確認済み）。
- **チームの資産が貯まる箱は、このgitリポジトリ ただ1つ。** 共有はgit経由（pull）でのみ起こる。
- **担当スキル（`/secretary` など）が動くのはClaude Codeだけ。** Projects・Cowork・claude.aiのチャットには自動で出てこない。
- だから **正本（箱）はgit一択**。Notionは「人が読むための窓（ミラー）」に徹し、正本にしない。箱が2つに割れた瞬間に「どっちが最新？」が始まり、資産は貯まらない。

### 3層モデル（役割を固定する）

| 層 | 何をする場所 | ツール | 主に誰が |
|---|---|---|---|
| 🏛️ 倉庫（正本） | スキル・CLAUDE.md・ナレッジ・成果物が貯まる唯一の箱 | **gitリポジトリ（このbalencer）** | 全員 |
| 🔧 作業台 | 調べ物・下書き・Notion/Drive読み書き・雑務の自動処理 | **Cowork**（Notion・Google Workspace・GitHub接続済） | 主にメンバー |
| 📖 窓 | 倉庫の中身を人が読む・タスクを見る | **Notion＋Projects** | 全員（人向け） |

原則：**Cowork/Notionは「倉庫に入れる前の下ごしらえ」。作ったものは必ず倉庫（git）に戻す。**

### 肝：ターミナルなしで同じ倉庫に入る＝デスクトップアプリの「Code」タブ
メンバー2人はターミナルに住まない。それでも同じ倉庫で作業できる方法が Claudeデスクトップアプリの **Code タブ**。
- チャットで指示 → 変更は画面で**差分レビュー** → **ボタンでコミット/プッシュ**（gitコマンドはClaudeが代行、本人は打たない）
- 開いた瞬間 CLAUDE.md も `/secretary` などのスキルも全部そのまま使える（阿部さんと同一環境）
- 必要な初期設定は一度だけ（各Macに git と gh を入れて clone）。以降は視覚的なCodeタブだけで完結。

### 資産が貯まるループ（金曜に固定）
利用の山が金曜。そこに週次棚卸しを置く。
```
平日：各自 Codeタブ or Cowork で作業
   └ 学び・決定 → docs/_knowledge/<担当>.md
   └ 繰り返す作業 → .claude/skills/（スキル化）
金曜：週次棚卸し（後述の3回ルールで昇格判定）→ commit & push
翌週：全員 pull → 全員が新スキル・新ナレッジを使える
```

### 3回ルール（阿部さんの既存ルールを維持・チームに拡張）
- **同じ手順が3回以上出てきたら、顧客に依存しない「スキル」に昇格**（CLAUDE.md／メモリの既存ルールそのまま）。
- チーム版：金曜の棚卸しで各自「今週3回以上やった作業は？」を出し合い、昇格するものを決める。
- 昇格作業（`.claude/skills/` の作成・更新）は当面 阿部さん（または慣れたメンバー）がCodeタブで行いコミット。メンバーは「これスキルにしたい」を金曜に挙げるだけでよい。

### ログイン/請求ルール
- **会社の仕事は全員チームアカウントで統一。** 会社利用がチーム側に集約され、管理者が利用状況（claude.ai/analytics/claude-code）を見られる。
- 個人で契約しているMaxプランは各自の私用のみ（会社の仕事には使わない）。1セッション内で請求先は選べないため、混ぜない。

### ブランチ運用（3人・軽量）
- mainで直接作業しない（CLAUDE.md 5章）。各自 `claude/<名前>-<用件>` ブランチで作業。
- ドキュメント類（docs配下）はプッシュ後に阿部さんがmainへマージ、または低リスクなものは各自マージ。3人なので重いレビュー体制は敷かない。

### 却下した案とその理由
- 「Notion／Projectsを起点にして成果物をそこに貯める」→ **却下**。スキルがClaude Codeでしか動かず、正本がgitと分裂するため。Notionは窓に限定。
- 「メンバーにターミナル操作を覚えてもらう」→ **不要**。デスクトップCodeタブで代替でき、学習コストを倉庫参加の条件にしない。
