## 大前提：必ず「balencerフォルダ」を開いた状態で話す

- 作業フォルダ: `/Users/abetakayuki/Projects/balencer`
- このフォルダを開くと `CLAUDE.md` が自動で読まれ、**最初から秘書として受付**します

## 1. CLI（ターミナルのClaude Code）

```bash
cd /Users/abetakayuki/Projects/balencer
claude
```

呼び出し方（どれでもOK）:
- そのまま話す: 「今日やること整理して」「○○社のミーティング準備して」→ 秘書が受付し、必要なら担当へ振り分け
- 秘書を明示: `/secretary`
- 担当を直接指名: `/dev` `/marketing` `/finance` `/creative` `/consulting`

## 2. VSCode拡張のCLI（この環境）

- VSCodeで **balencerフォルダをワークスペースとして開く**（File → Open Folder → `/Users/abetakayuki/Projects/balencer`）
- Claude Codeのパネル（チャット）を開く
- フォルダが開けていれば `CLAUDE.md` が読み込まれ、そのまま話すだけで秘書が受付。`/secretary` `/dev` 等のスラッシュ呼び出しも同じ
- ポイント: **balencerをワークスペースで開いていること**。別フォルダを開いていると秘書は出ない

## 3. デスクトップアプリ（Mac）

- アプリ起動 → プロジェクト（作業フォルダ）として `/Users/abetakayuki/Projects/balencer` を開く
- あとはCLIと同じ

## 4. クラウド / Web（claude.ai/code）

- リポジトリ `balencer-inc/balencer` を選んでセッションを開く（通常はmainブランチ）
- 開いた瞬間から同じ秘書・担当・ブランド定義で動く（各PCのグローバル設定が無くてもOK）

## 道具の役割分担（どれに何を置くか）

会社の仕事は複数の道具にまたがる。迷ったらこの境界線に従う。

| 道具 | 一言でいうと | 何を置くか | 共有範囲 | 正本？ |
|---|---|---|---|---|
| **GitHub（balencerリポジトリ）** | 会社脳・正本の金庫 | 会社/サービス定義、提案・見積・LP、AI社員スキル、ナレッジ | チーム全員 | ◎ 会社の唯一の正本 |
| **Claude Code** | 実行環境（AI社員が働く机） | ここには保存しない。動いて成果物をGitHub/Driveへ書き出す | — | ✕（道具） |
| **Google Drive** | 共有の作業場・原本置き場 | 営業/財務シート、売掛金管理表などExcel原本、メンバー共有の現物 | チーム＋一部クライアント | △ 数字系の原本 |
| **Notion** | 動的DB・追客・ネタ帳 | リードDB、SNSネタ帳、サービス資料ページ、企業ドック | 用途により共有/個人 | △ 運用データの倉庫 |
| **Obsidian（個人Vault）** | 個人脳・非共有 | 阿部さん個人の思考・判断基準・声のトーン、人に見せない私的な数字 | 阿部さんのみ（ローカル） | ◎ 個人の正本 |

**境界線（迷ったらこれ）**
- 会社・クライアントの仕事の精度 → GitHub ／ 数字の原本・共有現物 → Drive
- 溜め続ける運用データ・追客・ネタ → Notion
- 「阿部さん個人」の思考・見せたくないもの → Obsidian（＝共有側には置かない）

> Obsidianは非共有の個人脳。中身はここには書かない。会社側が守るのは「私的な思考・数字を共有リポジトリ/Drive/Notionに漏らさない」という境界だけ。Obsidian自体は「毎日インプットを投げるInbox＋週1でClaudeに処理させる」運用で育てる（詳細ルールは個人Vaultの `CLAUDE.md` に記載）。

## 運用の約束（CLAUDE.mdに明記済み）

- 作業はこのフォルダで（別フォルダだと秘書もブランド定義も出ない）

- タスク完了時はコミット＆プッシュまで（会話だけで終わらせない）
