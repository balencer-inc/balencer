## 第1部：オイシーズって、どういう会社？（中学生向け解説）

### ざっくり一言でいうと
**「有名なラーメン屋さんや天丼屋さんを、いくつもまとめて持っている会社」**です。自分たちで店をやるだけでなく、その「おいしいブランド」を他のお店に貸し出して、世界中に広げようとしている、勢いのある会社。

### 持っている代表的なお店（ブランド）
- **金子半之助（かねこはんのすけ）**：天丼・天ぷらの有名店。海外でも人気。
- **つじ田**：濃厚な魚介系スープのつけ麺の有名店。
- **田中商店 / 田中そば店**：博多長浜ラーメン・中華そば。

### この会社の「すごいところ」= 3つの特徴

**特徴1：コロナ後に売上が2倍になった急成長企業**
売上は50億円 → 100億円超へと、コロナ明けで倍増した。今まさに伸びている真っ最中。

*(中学生向け：会社の「稼ぎ」が数年で2倍。部活でいえば、去年まで地区大会だったのに今年いきなり全国レベル、みたいな急成長中ということ)*

**特徴2：海外にどんどん出ている（ここが超重要）**
海外の店の比率が約24.5%＝**4店に1店は海外**。タイ（財閥サハ・グループと組む）、インドネシア、フィリピンなどにFC（フランチャイズ）で出店を加速中。

*(中学生向け：FC＝フランチャイズとは「ブランドの名前と作り方を、現地のお店にお金をもらって貸すやり方」。コンビニと同じ仕組み。オイシーズは、日本の味を海外の会社に「のれん分け」して広げている)*

**特徴3：ブランドシェアリングという珍しい商売**
これがオイシーズ独自の面白い商売。**「今やっている別のお店（居酒屋・カフェ・定食屋など）に、つじ田などのメニューを"追加"で出させてあげる」**サービス。
- 材料は冷凍・パウチで届くので、**調理は1〜3分**、人を増やさなくていい。
- お店は「空いている時間・厨房」を使って、デリバリー等で追加の売上を作れる。
- オイシーズは店を作らずにブランドを広げられる（＝店舗を建てる大金が要らない）。

*(中学生向け：たとえば夜だけの居酒屋が、昼の空いてる時間に「つじ田のつけ麺」を作って売れる。居酒屋は儲かる、オイシーズもブランド料が入る、両方トク。これが「シェアリング＝共有」の意味)*

### 組織のかたち（ここに大事なヒントがある）
オイシーズは「持株会社（ホールディングス）」で、ブランドごとに**別会社**になっている。
- 金子半之助 → 株式会社バイザ・エフエム
- つじ田 → 株式会社H.I.T WORLD
- 田中商店 → 株式会社GMS
- グループ全体で**5社・約1,600名規模**。

*(中学生向け：1つの会社に全部入っているのではなく、ブランドごとに「子会社」に分かれている。野球でいうと、1つの球団の中に投手部門・打撃部門が別チームになっているような感じ。→ だから情報や書類も"バラバラの場所"にしまわれている可能性が高い＝後で重要)*

### 働く人の特徴
- **外国籍社員の採用比率28.9%**（≒3人に1人が外国籍）。特定技能スタッフも活躍。
- 「一人ひとり違っていい」を掲げてDEIB（多様性）を推進。
- OJT（現場で教える教育）と家族的なカルチャーを大事にする社風。

## 第5部：ゼロベースの解決策（バレンサーの都合を外した"正しい順番"）

もしオイシーズの経営だけを考えるなら、投資対効果の序列はこうなる：

1. **①発注・シフト（HANZO導入）** — 粗利に一番効く。ただしバレンサーの領域ではない → 素直にHANZOへ。
2. **⑤海外FCオペレーションAI** — 成長エンジンの海外を加速。経営インパクト最大級。← バレンサーの本命。
3. **⑥グループ横串ナレッジ / ③社内ナレッジAI** — 工数・教育・経営可視化。← バレンサーが伴走。
4. **④MVV浸透** — 一体感。社内主導＋AIに乗せる部分だけ伴走。

**ゼロベースの結論**：効果最大の①はバレンサーが取らない領域。だからこそ「①は御社が直接HANZOへ、我々は⑤⑥③で価値を出す」と序列ごと開示するのが最も誠実で、かつ"売り込まない右腕"としての信頼を最大化する。金額で差別化する余地は小さく、**勝負は⑤/②という誰もやっていない領域を描けるか一点**。

## 第6部：技術の道筋（複雑ではない、という事実）

### なぜ「巨大・複雑」ではないか
海外FCオペAIのコアは「RAG付きチャットボット1個」。2026年時点で枯れた構成。
- **多言語**：生成AI（Gemini/Claude）がネイティブ処理。「タイ語で聞かれたらタイ語で答えて」とプロンプトに書くだけ。翻訳エンジンの自作は不要。むしろ日本語専用より簡単。
- **マニュアル検索（RAG）**：NotebookLM / Vertex AI Search / Dify など既製基盤にドライブのマニュアルを流し込む。ベクトルDB自作は不要。ノーコード〜ローコード。
- **チャットUI**：LINE公式AIチャットボット(β)はPDFを入れるだけ。作り込むならMessaging API＋既製フレームワーク。定型。

### 必要な体制（スター人材は不要）
- **設計・業務理解・PM・クライアント伴走 → 阿部**（案件の心臓、外注不可、受注の根拠）
- **RAG/チャットボット実装 → 既存の外部開発パートナー or 中堅実装者1名スポット**
- **AIチューニング・プロンプト設計 → 阿部＋Claude Code等のAIツール**

### 段取り（一気に作らない）
Phase 0診断 → Phase 1で日本の本部PoC（1ブランド・日本語）で型を作る → その型をタイ1店舗へ横展開（言語はプロンプト＋マニュアル差し替え）→ 順次拡大。
**海外・多言語は"新規開発"ではなく"同じ型の横展開"**。最初の1本さえ握れば、あとは反復。

### 段取りで消せるリスク（未知の技術難関ではない）
- 現地チャットアプリ差異（タイ=LINE強い / インドネシア=WhatsApp優勢）→ コアAIは共通、入口アプリを差し替えるだけ。
- 本部マニュアルの品質不足 → Phase 0診断で炙り出す対象そのもの。むしろバレンサーの価値。
