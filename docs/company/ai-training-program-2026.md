# AI研修プロジェクト 商品設計（2026・叩き台v0）

> 目的: 「AI研修」を3人の旗艦自社プロジェクトにする。
> 位置づけ: これは集客商品である以上に **「最大顧客依存（睦備/むっちゃん 約60%）からの脱出エンジン」**。
> 由来: 2026-06-25 阿部×加藤（梨紗）相談で出た軸。詳細経緯は `org-design-2026.md` ⑤章・`_knowledge/consulting.md`。
> ステータス: 叩き台。3人ミーティングの土台にする。確定したら正式版へ。

---

## 0. なぜこれをやるのか（3行で）

1. 売上の約60%が睦備1社。1社飛ぶと体制が組めない＝**生存リスク**。
2. 人脈外の新規がわずか3.7%。**人脈に頼らない新規エンジンが要る**。
3. AI研修なら——新規リードを生み、型化でき、3人で協業でき、かおりん（松本）の格上げとAI習熟も同時に進み、**自社がAI組織OSの実証物になる**。一石で複数を獲る。

---

## 1. 商品の骨格 ＝ 単発で終わらせない

研修は **入口（ENTRANCE）**。研修単体で完結＝単発受注になり看板2枚から切れる。最初から動線を設計する。

```
広告 / SNS / 紹介
      ↓
  AI研修（入口・低単価 or 無料〜数万）
      ↓
  AI活用診断（研修内で自社の課題を可視化）
      ↓
  個別相談（ミタテ／SOERU提案）
      ↓
  伴走（CORE：Main右腕 → Premium → 組織開発/AI組織OS）
```

- **看板2枚への接続**: 研修で「AIで会社の戦い方が変わる」を体感 → 組織開発・AI実装の伴走へ。
- **既存サービス階段との接続**: 研修 → SOERU/ミタテ（入口商品）→ Main → Premium → AI組織OS。営業戦略 `sales-strategy-2026.md` 4-4のセミナー/広告ファネルの「テーマを"AI研修"で具体化した版」。

---

## 2. 研修の中身（叩き台・要3人で詰める）

> ここは仮置き。ターゲット・テーマ・単価は§6の未決事項で確定する。

- **誰に**: 地域オーナー企業の経営者・幹部（主軍セグメント）。「うちもAIを、と思っているが何から手をつけるか分からない」層。
- **約束する変化**: 「AIで明日から自社の業務が1つ軽くなる」＋「会社の戦い方を変える絵が見える」。
- **形式（案）**: 90〜120分のセミナー型 ＋ ワーク（自社業務をその場でAIに渡してみる）。オンライン/会場どちらも。
- **持ち帰り**: 自社のAI活用診断シート1枚（＝個別相談の入口）。
- **強み**: 「外注の7割をAIで回す自社」をそのまま見せられる＝生きた実証（他社が持てない武器）。

## 3. 3人＋AI社員の分担（専門性を立てる）

| 担当 | この PJ での役割 |
| --- | --- |
| **阿部** | 入口創出（外部交流・登壇先・過去顧客への案内）、研修の方針・芯づくり。リアル営業と接続 |
| **梨紗（加藤）** | 研修の提案・登壇、参加者からの個別相談クロージング。後工程はAI＋かおりんへ寄せる |
| **かおりん（松本）** | **自社プロジェクトの実働主**: LP制作・広告クリエイティブ・広告出稿・申込〜追客の運用。ここでAIを使いこなす過程を仕組み化（＝AI組織OSの自社モデルケース） |
| **AI社員** | dev=LP実装、marketing=広告/SNS下書き・追客、creative=クリエイティブ叩き、consulting=研修コンテンツ/診断シート叩き、secretary=申込管理・連絡 |

> ねらい: 「提案が梨紗に集中」「かおりんが薄い」を、この PJ の分担で同時に解く。かおりんは"使う場"を得てAI習熟、梨紗は後工程を手放す。

## 4. ファネル（かおりんが回せる粒度）

| 工程 | 中身 | 主担当 |
| --- | --- | --- |
| 1. 集客 | 広告（Meta）／SNS／既存・休眠への案内 | かおりん（広告）＋阿部（案内） |
| 2. LP | 研修申込LP（安っぽくしない・看板2枚のトーン） | かおりん＋dev |
| 3. 申込 | フォーム＋自動返信（LINE不使用） | かおりん＋secretary |
| 4. 研修 | セミナー実施＋AI活用診断 | 梨紗（登壇）＋阿部 |
| 5. 相談 | 個別相談→ミタテ/SOERU提案 | 梨紗（クロージング） |
| 6. 伴走 | CORE（Main→Premium→AI組織OS） | 阿部＋梨紗 |

- **広告は検証順守**（営業戦略4-4）: いきなり予算投下しない。①広告ゼロで無料開催→転換率検証 ②転換が立てば月5〜10万から小額テスト。

## 5. このPJが動かす数字（KPI紐付け）

| 会社KPI | 現状 | このPJの寄与 |
| --- | --- | --- |
| 最大クライアント依存 | 約60% | 睦備以外の新規受注で **60%→まず40%** |
| 人脈外の新規比率 | 3.7% | 広告・SNS経由の新規で **→20%超** |
| かおりんのAI習熟 | 低 | LP/広告/出稿を実務で習得＝AI組織OS実証 第1号 |

PJ自体のKPI（叩き台）: 申込数 → 出席率 → 個別相談転換率 → 受注数。まず「広告ゼロ1回開催」で後半の転換率を測る。

## 6. 未決事項（3人で決める）

1. **ターゲットの一本化**: 経営者向け / 現場向け / 同業（制作・システム会社）向け、どれを最初の的にするか。
2. **研修テーマ**: 「経営者のためのAI入門」か「業務を1つ自動化する実践」か等。
3. **単価**: 無料（リード獲得特化）／数千〜数万（質の高い見込み客に絞る）。※価格は阿部さん最終判断。
4. **初回開催の時期**: 営業戦略では8月に第1回無料セミナー想定 → これをAI研修として走らせるか。
5. **AIツールのチーム共有**: 今は不要（PJで覚える優先）。本当に必要になったらClaude Teamプラン検討。
