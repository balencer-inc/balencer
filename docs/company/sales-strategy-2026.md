## 1. 結論サマリ

**4つのチャネルは「全部やる」。ただし役割と順番がそれぞれ違う。**

| チャネル | 役割 | 今期の扱い |
|---|---|---|
| 紹介・休眠・対面（その他） | **今期の売上をつくる主戦場** | 6月から全力。阿部さんの時間の60% |
| 補助金フック | チャネルではなく**商談のクロージング武器** | 対面商談に組み込む。看板には出さない |
| SNS発信 | **商談カレンダーを埋める上流**（認知・第一想起・休眠を温める） | 6月から型どおり継続。週3〜4 |
| 広告経由ファネル | **来期の新規エンジンの試作** | 今期は準備→無料検証→小額テスト。いきなり予算投下しない |

捨てるチャネルはない。捨てるのは「全部を同時に全力でやること」。
合わないのは唯一「今すぐ広告に予算を流すこと」——出口商品（SOERU再設計版）・LP・セミナーが未検証のうちに広告費を使うと無駄打ちになるため、検証の順番を守る。

## 3. 前提（再定義との接続）

- 看板は2枚のみ: **①組織開発・MVV ②AI実装**。営業トークも発信もこの2枚から出ない（docs/handoff/ 正本）
- 階段: SOERU（入口・月10/20万）→ Main → Premium → AI組織OS/承継
- **補助金商品群はチャネルであり看板に出さない**（2026-06-11 再定義で確定）
- 2026-06-09 決定済み: 阿部さんは「対面フェーズ」に入る。売上の73%が人の信頼由来であり、対面時間は信頼が最速で立つ相手（紹介・既存・休眠・温リード）に集中する
- 実装・納品はメンバーにほぼ任せられる＝阿部さんの時間を営業に振れる

### 商品導線の原則: 新規はSOERUが顔、組織開発はバックエンド（2026-06-12追記）

組織開発・MVV策定は初見の経営者には伝わりにくい（無形・効果が見えるまで時間がかかる・比較対象がない）。そこで:

- **新規獲得チャネル（広告・セミナー・SNS経由の新規・補助金フック）の商品導線は、SOERUとAI実装系（ミタテ等）を前に出す。** 「月額で雇えるAI右腕」「数字が見える化される」は初見でも価値が分かる
- **組織開発・MVVはバックエンド。** SOERU・ミタテで毎月の接点ができ、課題が数字の外側（組織・人・理念）にあると互いに見えてきた段階で提案する。階段: SOERU/ミタテ → Main → MVV策定・組織開発 → AI組織OS/承継
- 例外: **信頼が既にある相手（紹介・既存・休眠）には最初からMVVを出してよい。** トライ・タス・ハリマの実績が根拠として使えるのはこのルートだけ
- 看板2枚（組織開発MVV×AI実装）はHP・ブランドの構えとしてそのまま維持する。これは「会社の旗」と「新規導線の入口商品」の役割分担であり、再定義との矛盾ではない

## 4. チャネル別戦略

### 4-1. 紹介・休眠・対面 ＝ 主戦場

今期の受注はここから出る。新しい仕組みの立ち上がりを待たない。

- **最優先: 紹介連鎖9社の起点だった休眠先（美容室グループ・F社）の再訪**。紹介エンジンの再起動が単発受注より価値が高い
- 休眠掘り起こし上位・紹介ハブの実名リストと優先順位は docs/sales/ 管轄（機密）
- 商談の持ち物: 看板2枚の説明（docs/handoff/pages/）＋ ミタテ補助金パッケージ（4-2）＋ SOERU入口
- cold（面識ゼロ）の対面営業はやらない。未経験の勝負は勝ち筋（紹介・休眠）で成功体験を作ってから

### 4-2. 補助金フック ＝ クロージングの武器

**位置づけが最重要: 補助金は「集客の看板」ではなく「対面商談で背中を押す最後の一押し」。**
看板に掲げると価格目当ての客が集まり、再定義（補助金＝チャネル）と矛盾する。

#### 登録状況と前提（2026-06-12時点）
- ミタテ（経営ボード）: **ソフトは登録済み**。役務（導入設定・カテゴリー6）と保守（カテゴリー7）は審査中
- 価格構造: ソフト150万固定＋役務伸縮で **150〜295万円（補助1/2 → 顧客実質75〜147.5万円）**
- スケジュール: ツール登録は随時。通常枠1次交付申請は6/15締切（今からは実質間に合わない）→ **2次以降の交付申請枠を狙う**。2次の締切日は事務局公表を確認すること（6月の宿題）

#### 役務・保守も登録された場合（本線）
1. 対面商談（4-1）で経営の数字まわりの課題が出たら、ミタテ150〜295万パッケージを提示。「補助金で実質半額」はクロージング段階でのみ使う
2. 導入時のNotion構築は**最小限（売上・会計の受け皿DBのみ、役務内30万/30hが上限）**。作り込むとヒトツが売れなくなる
3. 導入2〜3ヶ月後、「数字集めが手間」と顧客が自覚したタイミングで**ヒトツを別契約・別見積で提案**（アドオン逆ルート）。同一見積もりに混ぜるのは実質値引き扱いで規約違反リスク、絶対NG
4. 出口は月額伴走（SOERU→Main）へ接続。補助金は入口、本体は伴走

#### 役務・保守が不採択だった場合（保険）
- ソフト単体（150万・補助75万）でも提案は成立する。導入支援は補助金外の別建て見積に切り出す
- 修正を繰り返して通らない場合は構造を疑う（AIシャインの教訓）。ソフト単体運用に早めに切り替え、無理な再申請で時間を溶かさない

#### 数字の目安
- 補助金フック経由の今期目標: **提案5件 → 交付申請2〜3件 → 受注2件**
- ヒトツへのブリッジが決まれば1社あたり初年度1,000万円超・3年LTV 2,500万円規模（検算済みシミュレーションは docs/products/hojokin-2026/ 配下）

### 4-3. SNS発信 ＝ 空中戦（上流）

方向性はNotion「SNS発信阿部方向性」＋運用の型（2026-06-12策定）で確定済み。**このチャネルは今のままで合っている。**戦略上の位置づけだけ明確にする:

- **Xの役割**: 「うちもAIを、と思ったらまずバレンサー」の第一想起づくり。3柱（AI実装のリアル知見／ブランディング×AI／承継×AI）×逆三角形×長文7割
- **Facebookの役割**: 休眠・既存・紹介元に「阿部さん最近面白いことやってるな」を思い出させる。**3-1の対面営業の地ならし**であり、Xとは別物。語り直し・CTAなし・週1〜2
- 直近タスク: プロフィール改修（Bio→名前欄→固定投稿→リンクの順、案は策定済み）
- 成果は相談・DM・休眠からの返信数で見る。インプレッションは追わない
- 3ヶ月やって相談導線ゼロなら、投稿の質ではなく柱の配分かプロフィール構造を疑う
- 運用体制: マーケティング担当スキル（~/.claude/skills/marketing/）が下書きを生成し、阿部さんは手直しと投稿のみ。毎週月曜朝に下書きセット（X2本＋FB1本＋一次情報3問）を自動生成するルーティンで回す。自動投稿はしない（投稿ボタンは常に阿部さん）

### 4-4. 広告経由ファネル ＝ 来期エンジンの試作

8工程（広告→クリエイティブ→LP→セミナー誘導→受け皿→スライド→追客→クロージング）の設計とツールスタックは確定済み。受け皿はフォーム＋Notion（LINE不使用）、追客は apps/outreach 転用。

**ただし、広告費を流すのは検証が済んでから。** 理由は3つ:
1. 出口商品（SOERU再設計版）の名称・価格がまだ確定していない。出口が曖昧なファネルに広告費を入れても回収できない
2. CI v3・看板ページが制作中。LPのコピーとトーンが固まる前に作ると二度手間
3. 阿部さんの時間は対面60%に振っており、セミナー登壇を含むファネル運用を今すぐ重ねるとどちらも中途半端になる

**段階導入の3ステップ:**
- **Step 1（6〜7月・準備）**: SOERU確定 → セミナースライドのbake-off（Claude Code vs Manus）→ セミナーLP＋受け皿フォーム＋追客テンプレ
- **Step 2（8月・広告ゼロ検証）**: 第1回無料セミナーを**広告なし**で開催。集客はSNS・既存・休眠への直接案内のみ。これでコンテンツ・申込率・商談転換率というファネル後半を広告費ゼロで検証できる
- **Step 3（9月〜・小額テスト）**: Step 2の転換率が立っていればMeta広告を月5〜10万円から。数字が立たなければ広告ではなくセミナー内容を直す

セミナー設計の原則: 価値提供8割＋出口案内2割。24時間オファーは使うが、カード自動引き落とし1年ロック型は信頼毀損リスクがあるため、請求書・年間契約の見せ方にする（SOERUは最低3ヶ月）。

## 6. KPI（月1ふりかえり）

- 阿部さんの商談数／月（紹介・既存・休眠・温リードの内訳）
- 紹介発生数、休眠からの返信数
- 補助金: 提案件数 → 交付申請件数 → 受注件数
- SNS: 相談・DM・休眠返信数（インプレッションは参考値）
- セミナー: 申込数 → 出席率 → 商談転換率（Step 3判断の根拠）
- 構造指標: 人脈外の新規比率 3.7% → 20%超／最大クライアント依存 40% → 25%以下

## 7. やらないことリスト

1. 補助金を看板・広告コピーの主役にしない（クロージングでのみ使う）
2. ファネル後半が未検証のうちに広告予算を投下しない
3. cold対面営業に時間を使わない（信頼が最速で立つ相手に集中）
4. ミタテ導入時にNotionを作り込みすぎない（役務内30万/30h上限。ヒトツの余地を残す）
5. ヒトツと補助金ツールを同一見積もりにしない（実質値引き扱いの規約違反リスク）
6. LINEを受け皿にしない（BtoB月10〜50万円帯ではチープに映る）
7. 看板2枚以外のサービスを自己紹介・営業トークに出さない

## 8. 見直しの条件

- 役務・保守が不採択 → 4-2の保険ルートへ即切り替え（再申請の修正ループに入らない）
- 8月セミナーの商談転換がゼロ → 広告は凍結し、出口商品かセミナー内容を疑う
- SNS3ヶ月で相談導線ゼロ → 柱の配分とプロフィール構造を見直す
- 対面商談が月の予定を埋め切った → 広告ファネルの優先度をさらに下げてよい（嬉しい誤算）
