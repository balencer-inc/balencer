# 第1回 講師用スライド内容（テキスト版）
## AIの全体像と主要ツールの使い分け

> AI実装研修（リスキリング助成金）／全5回の第1回・OFF-JT 3時間
> 対象：アナログ中心の会社の実務チーム（IT専任なし前提）。専門用語は避け、出たら必ず言い換える。
> この資料は「スライドの中身」をテキストで確定したもの。これを元に後でスライド化する。
> 進め方の基本：1スライド＝1メッセージ／座学は短く・手を動かす時間を長く／「失敗してOK」の空気を最初に作る。

**第1回のゴール（受講後にこうなっている）**
1. AIが「何が得意で・何が苦手か」を自分の言葉で説明できる。
2. ChatGPT・Claude・Gemini の違いと使い分けが分かる。
3. やってはいけないこと（情報の扱い）を理解し、安心して触れる。
4. 実際に自分の仕事で1つ、AIに頼んでみた経験がある。

**3時間の時間配分（休憩2回・各5〜10分）**
| 区切り | 内容 | 目安 |
|---|---|---|
| 0. オープニング | つかみ・ゴール・進め方・安心設計 | 15分 |
| 1. AIの全体像 | 生成AIとは／得意・苦手／なぜ今／業務での効き方 | 30分 |
| ―休憩― | | 5分 |
| 2. 主要ツールの使い分け | ChatGPT/Claude/Gemini/Copilot・当社はClaude軸の理由 | 30分 |
| 3. 安全な使い方・社内ルール | 情報漏れ・うそ(ハルシネーション)・社内ルール | 30分 |
| ―休憩― | | 10分 |
| 4. ハンズオン「まず触ってみる」 | アカウント／最初の頼み方／自分の仕事で1つ | 45分 |
| 5. まとめ・宿題・次回予告 | 振り返り・宿題 | 15分 |

## 1. AIの全体像（30分）

### スライド5 ｜ そもそも生成AIって何？（むずかしくなく）
**画面に出す**
- 「ものすごく物知りで、文章を作るのが得意な"超優秀な新人さん"」
- 言葉で頼むと、言葉で返してくれる（文章・要約・翻訳・アイデア・下書き）
**講師が話す**
- 専門用語（大規模言語モデル等）は使わない。「新人さん」の比喩で通す。
- 「検索」との違い：検索は"探す"、AIは"作る・考える手伝い"。

### スライド6 ｜ AIが得意なこと
**画面に出す**
- 文章を書く・直す（メール、報告書、議事録の下書き）
- 要約する（長い文・資料を短く）
- 翻訳・言い換え（やわらかく／かたく）
- 案を出す（タイトル、企画、たたき台）
- 整理する（バラバラの情報を表に）
**講師が話す**
- それぞれ受講者の業務に寄せた例を口頭で。「みなさんの仕事だとこれ」。

### スライド7 ｜ AIが苦手なこと（ここ大事）
**画面に出す**
- 最新の出来事・社内だけの事情は知らない
- ときどき"それっぽいウソ"を自信満々で言う（あとで詳しく）
- 正確な計算・大量データの厳密処理は不得意な場面がある
- 「察して」が苦手＝具体的に頼むほど良くなる
**講師が話す（ねらい）**
- 苦手を先に知ると、過信せず・がっかりせず付き合える。

### スライド8 ｜ なぜ"今"AIなのか
**画面に出す**
- 数年で精度が一気に上がり、誰でも言葉で使えるようになった
- 人手不足のなか「一人ひとりの生産性」を上げる現実的な手段
- 競合も使い始めている＝差がつき始めている
**講師が話す**
- あおらず事実ベースで。「特別な人のものではなくなった」。

### スライド9 ｜ 会社の仕事にどう効くか（イメージ）
**画面に出す**
- 例：問い合わせメールの下書きが数分→数十秒
- 例：長い議事録の要約が即時
- 例：Excelの整理・転記の相棒
- ※「置き換え」ではなく「下ごしらえを任せる」発想
**講師が話す**
- 第3〜4回でこれを実際にやる、と予告して期待をつくる。

## 2. 主要ツールの使い分け（30分）

### スライド11 ｜ 主要なAIは大きく3つ（+1）
**画面に出す**
- ChatGPT（OpenAI）／Claude（Anthropic）／Gemini（Google）／（+Copilot＝Microsoft）
- どれも「言葉で頼む」のは同じ。得意分野と"クセ"が違う
**講師が話す**
- 「全部覚えなくていい。違いの"ざっくり"だけ」。

### スライド12 ｜ それぞれの得意分野（ざっくり）
**画面に出す**
- ChatGPT：万能・利用者が多い・情報や連携が豊富
- Claude：長い文章の読み書き・丁寧で自然な日本語・じっくり考える作業／"自分で仕組みを作る"のに強い
- Gemini：Google（Gmail・ドキュメント・スプレッドシート）と相性が良い
- Copilot：WordやExcelなどMicrosoft製品の中で使える
**講師が話す**
- 1社をけなさない。「適材適所」。受講者の利用環境（Google/Microsoft）を質問して結びつける。

### スライド13 ｜ この研修で"軸"にするのはClaude（理由）
**画面に出す**
- 日本語が自然で、長い資料も任せやすい
- 手作業の自動化・小さなアプリ作りまで踏み込める（第3〜4回で使う）
- だからこそ「使えるところまで」到達しやすい
- ※他のAIも紹介する。最後は自社に合うものを選べばよい
**講師が話す（ねらい）**
- 「押し売りではなく、実務で一番遠くまで行けるから軸にする」と正直に。

### スライド14 ｜ 使い分けの考え方（迷わないコツ）
**画面に出す**
- まずは1つを"いつもの相棒"にして慣れる（本研修ではClaude）
- 普段使うツール（Google/Microsoft）の中で完結したい時は、それに合うAIも併用
- 大事なのは「ツール選び」より「頼み方」（第2回でやる）
**講師が話す**
- ツールジプシーにならないこと。慣れが一番効く。

## 3. 安全な使い方・社内ルール（30分）

### スライド15 ｜ 安心して使うために、3つだけ守る
**画面に出す**
- ①出してはいけない情報を入れない ②答えをうのみにしない ③会社のルールに沿う
**講師が話す**
- 「怖いから使わない」ではなく「ここだけ守れば安全に使える」へ。

### スライド16 ｜ ①情報の扱い（一番大事）
**画面に出す**
- 入れてよい：一般的な相談、社外に出してよい文章、自分で考えた内容
- 注意：顧客の個人情報、未公開の数字、機密、パスワード等は安易に入れない
- 迷ったら「これ、社外の人に見せて大丈夫？」で判断
**講師が話す（ねらい）**
- 具体例で。法人向け設定では学習に使われない場合がある等は触れつつ、まずは"原則入れない"を徹底。

### スライド17 ｜ ②"それっぽいウソ"に注意（ハルシネーション）
**画面に出す**
- AIは知らないことも、自信満々にもっともらしく答えることがある（＝ハルシネーション＝AIの作り話）
- 対策：事実・数字・固有名詞は必ず人が確認／出典を聞く／重要判断は最終的に人
**講師が話す**
- その場で1つ"ありそうなウソ"を実演できると刺さる（任意）。

### スライド18 ｜ ③社内ルールの作り方（たたき台）
**画面に出す**
- 使ってよい業務／だめな業務の線引き
- 入れてよい情報／だめな情報の例
- 困ったら誰に聞くか（社内の窓口）
- ※完璧でなくてOK。まず簡単な1枚から始める
**講師が話す**
- 「ルールが無いと現場が止まる。今日のたたき台を持ち帰って育てましょう」。

### スライド19 ｜ ミニ確認クイズ（3問）
**画面に出す**
- Q1 顧客の個人情報を入れて要約させてよい？
- Q2 AIが出した数字はそのまま資料に使ってよい？
- Q3 使い方に迷ったらどうする？
**ねらい**
- 手を挙げる/口頭で。理解の定着＋場が和む。

## 4. ハンズオン「まず触ってみる」（45分）

### スライド21 ｜ まずは画面を開く
**画面に出す**
- 開く場所（URL）／ログイン／チャット欄はここ、と画面で指さし
- 「メッセージアプリに話しかける感覚でOK」
**進行メモ**
- 全員が同じ画面に着くまで待つ。置いていかない。机間巡視。

### スライド22 ｜ 最初の1打：あいさつしてみる
**画面に出す**
- 例：「こんにちは。あなたに何ができるか、3つ教えて」
- まず"返ってくる体験"をする（成功体験）
**講師が話す**
- ここで全員に「返ってきた！」を体験させるのが目的。

### スライド23 ｜ 仕事で1つ頼んでみる（型を渡す）
**画面に出す**
- 頼み方の型：「①誰として ②何を ③どんな形で」
- 例：「飲食店の店長として、来店のお礼メールを、やわらかい口調で200字で書いて」
- 自分の"任せたい面倒"で1回やってみる
**進行メモ**
- 3〜4人に画面共有/発表してもらう。うまくいった例・いまいちな例の両方を拾う。

### スライド24 ｜ もう一歩：直してもらう
**画面に出す**
- 「もっと短く」「やわらかく」「箇条書きで」と追加で頼める（会話が続く）
- AIは"一発"でなく"やりとりで仕上げる"
**講師が話す**
- ここが第2回（頼み方の型）への橋渡し。

### スライド25 ｜ うまくいかない時のコツ
**画面に出す**
- 具体的に頼む（誰に・何を・どんな形で・長さ）
- 例を見せる／背景を伝える
- 一度でダメなら、追加で注文する
**講師が話す**
- 「AIが悪いのではなく、頼み方で9割変わる」。

## 備考
- 助成金（事業展開等リスキリング支援コース）の要件上、本回はOFF-JTとして実訓練時間に算入。休憩・移動はカウントしない（出席簿で実時間を記録）。
- スライド化する際は「1スライド1メッセージ・文字は最小・図と余白多め」を基本に（バレンサーCI＝白ベース×蛍光イエロー）。
