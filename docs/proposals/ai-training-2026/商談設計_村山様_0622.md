## 0. この商談の到達点（先に結論）

- **ゴール**：村山さんが「散らかっていた頭が整理された＝有益だった」と感じ、**まず小さく1歩（印税分配の要件定義）を発注**したくなる状態にする。
- **散らかりの正体**：村山さんは「やりたいこと5個」「補助金は使えるのか」「研修なのか作ってもらうのか」が **混ざったまま**。これを解くのがこちらの価値。
- **こちらの解き方（1枚で見せる）**：
  - 5テーマを **2レーンに仕分ける** → ①**研修で自走**できる業務 ／ ②**作ってお渡しする受託**が早い業務（＝本命の印税分配）
  - 制度は **正直に交通整理** → 印税分配はAI導入補助金の“外”。研修はリスキリング助成金で実質負担が小さい。
  - 本命の印税分配は **類似ケース（NAILIT同型）で「こう自動化できる」を具体で見せる**。
- **着地の最小単位**：契約を一気に取りに行かない。**「印税ステートメントのサンプルを2〜3種いただく → 要件定義 → 見積」** という “次の一歩” を合意して終わる。これが一番決まりやすい。

## 2. 5テーマの仕分け地図（商談の核・ホワイトボード代わりに見せる）

| # | テーマ | レーン | 出口 | 補足 |
|---|---|---|---|---|
| 1 | **印税分配の仕組化** | **②受託（作ってお渡し）** | **受託開発**（本命）。要件定義後に見積 | 難度・正確性要求が高く、研修で自走させるより**作った方が早い・確実**。NAILIT催事精算と同型 |
| 2 | ワークフロー仕組化・マニュアル化 | **①研修で自走** | リスキリング研修 ＋（必要なら）ヒトツの型 | チームがClaudeで自分の業務を仕組化できるようになる＝研修の主戦場 |
| 3 | メール仕分け＋返信下書き | ①寄り（一部②） | 研修で型を渡す／本格運用は受託 or SOERU | まず研修で「自分でできる」を体験 → 重ければ受託 |
| 4 | サイボウズ移管 | — | 引っ越し＋整理（スポット） | AI案件ではない。**やるなら別建て・小さく**。商談では深追いしない |

→ 5個の混乱が「研修1レーン＋受託1本＋後回し」に**収束する**。これが村山さんの“お土産”。

## 3. 制度の交通整理（正直さで信頼を取る）

村山さんは元々「**AI導入補助金でClaude研修を**」と言っていた。ここを正しく整理して見せる＝専門家としての信頼を一気に得るポイント。**脚色せず、使える/使えないをフラットに。**

| 制度 | 何に使えるか | 今回の当てはまり |
|---|---|---|
| **AI導入補助金（経産省）** | 登録済みソフトの“導入”が本体。研修だけは対象外 | **印税分配は対象外**（経営の数字の見える化＝別業務とプロセス不一致）。無理に乗せない |
| **リスキリング助成金（厚労省）**<br>人材開発支援助成金 事業展開等リスキリングコース | **研修そのもの**が対象。中小は経費**最大75%**＋賃金助成1,000円/h | **①研修レーンの受け皿**。実質負担を大きく下げられる |
| **受託開発（補助金なし）** | — | **②印税分配の本命**。要件定義 → 見積 |

**正直に言う前提（守る）**
- 申請主体は **1st PLACE（事業主）**。**申請代行は社労士の専権** → バレンサーはやらない／やれない。**渡せるのは研修の中身＋添付書類**（カリキュラム表・見積書・講師要件確認書・出席簿・実施報告書）。
- **計画届は研修開始の1ヶ月前まで**（1日遅れで全額不支給）→ 逆算すると6/22から最速で **8月以降の研修開始**が現実的。日程は焦らせず正しく引く。
- 制度は **2026年度（令和8年度）まで**。入金まで6〜8ヶ月。数値は2026/6時点の参考値・最終可否は労働局判断。

> 言い回し：「**研修への補助は“AI導入補助金”ではなく“リスキリング助成金”が正しい受け皿です。ここを間違えると不支給になるので、まず交通整理させてください**」

## 4. 類似ケースの出し方（本命＝印税分配を“想像できる”状態にする）

- **メインで見せる**：`case-study.html`（匿名事例＝NAILITの業務自動化を社名・金額伏せて橋渡し）。
  - 位置づけの一言：「**催事の精算で、複数フォームの明細を共通化して自動で割り当てる仕組みを作りました。御社の印税分配は“これと同じ型”です**」
  - 注意：「**Notion中心の例。御社はClaude中心で、裏のデータベースは適材適所**」と添える（Notion導入の話と誤解させない）。NAILIT実名資料は**社外に出さない**。
- **補足で触れる**：ワークフロー仕組化・マニュアル化＝「ヒトツ」の型（②テーマ2の出口）。深入りはしない、本命は印税分配。

**印税分配の“こう自動化できる”を具体で**（先方が頭で動く絵を渡す）：
> 複数社・複数フォームのステートメント（PDF/Excel）→ **共通明細に正規化** → 分配条件（権利者／楽曲／料率／按分）で **ルール化** → **自動計算＋根拠の追跡** → 支払通知書を発行。
→ 「ここまでが**作ってお渡しする範囲**。正確に作るには**実際のステートメントを2〜3種見せていただく**のが最短です」＝**次の一歩の発注**へ自然につながる。

## 5. 商談フロー（60分想定・時間配分つき）

| 時間 | パート | やること | 着地 |
|---|---|---|---|
| 0–5分 | アイスブレイク＋ゴール宣言 | 「今日は**やりたいことを整理して、進め方を一緒に決める**会にします」 | 期待値を“整理と意思決定”に置く |
| 5–15分 | **要件の読み合わせ**（→§1） | 5テーマを並べ直し、ズレを直す。法務は触らないと明言 | 「**正しく分かってくれている**」 |
| 15–25分 | **仕分け地図**（→§2） | 2レーン＋後回しに整理して見せる | 「**散らかりが解けた**」＝お土産① |
| 25–35分 | **制度の交通整理**（→§3） | 補助金/助成金/受託を正直に。研修はリスキリング | 「**信頼できる**」＝お土産② |
| 35–48分 | **本命＝印税分配**（→§4・デッキ＆事例） | 同型ケース＋自動化の絵 → 作ってお渡しする範囲を明示 | 「**できそう・任せたい**」 |
| 48–55分 | **進め方の合意**（→§6） | 次の一歩＝サンプル提供→要件定義→見積。並行で研修の段取り | **小さく発注の合意** |
| 55–60分 | 宿題と次回 | 先方：ステートメント2〜3種／こちら：要件定義の枠と見積準備 | 次回日程を仮押さえ |

**回す上での肝**
- **金額は先に言わない**（→§7）。流れ＝価値を理解 → 進め方 → 最後に費用。受託額は資料に出さず「要件定義後にお見積もり」。
- **全部やります、と言わない**。④⑤は「急ぎでなければ後で」と**こちらから引く**＝かえって信頼される。
- **即決を1個だけ作る**：「**まずステートメントを数種いただけますか**」。これは無料で・低負荷で・YESと言いやすい。商談を“前進した”状態で終える。

## 6. 着地＝発注の最小単位（散らからないための一本道）

```
【今日 合意するもの】
 第一歩：印税分配の「要件定義」へ進む
   → 先方の宿題：印税ステートメントのサンプル 2〜3種（社名伏せ可）
   → 当方：受領後に「できる/できない・難所」を即判定 → 要件定義 → 受託見積

【並行レーン】
 研修（リスキリング助成金）の段取り
   → 助成を使うなら社労士連携・計画届（開始1ヶ月前）を逆算
   → カリキュラムは samples/curriculum.html を叩き台に

【後回しでよいもの】
 サイボウズ移管 / Eight移管 → 本命が動き出してから
```

> 受託の本命像（社内メモ）：印税分配の構築＋保守。LTVは要件次第。**まず要件定義から入り、ここで信頼を作って受託本体へ**。月額の継続伴走が要る話になればSOERU（¥100,000/月〜）が受け皿。

## 9. 想定問答（刺さりやすい質問への返し）

- **Q. 補助金で全部できる？** → 「研修はリスキリング助成金で実質負担を下げられます。ただ**印税分配の“開発”は補助金の対象外**なので、ここは受託でお作りします。無理に補助金に乗せると不支給リスクが出ます」
- **Q. 申請は御社がやってくれる？** → 「**申請代行は社労士さんの専権**なので私たちはやりません。**研修の中身と必要書類は全部こちらで用意**し、社労士さんと連携します。丸投げにならないよう段取りはこちらで引きます」
- **Q. 研修だけで印税分配も自分たちでできる？** → 「ワークフローの仕組化は研修で自走できます。ただ**印税分配は正確性が命**なので、ここは作ってお渡しした方が早くて確実です。研修＝自走領域、受託＝お任せ領域、と分けるのがおすすめです」
- **Q. いつから動ける？** → 「研修で助成金を使うなら**計画届が開始1ヶ月前まで**。逆算で8月以降が現実的です。一方、**印税分配の要件定義は今日いただくサンプル次第ですぐ**着手できます」
- **Q. Eightの移管は？** → 「Eightは**エクスポート制限**があるので、できる範囲を先に確認させてください。ここは本命が動いてからで十分です」
